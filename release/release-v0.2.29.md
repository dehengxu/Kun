# Kun v0.2.29

这一版是 v0.2.28 之后的一次会话体验、项目协作与 Runtime 稳定性更新。重点是新增会话导出、运行时图片上传和生成图片轮播，加入项目级 MCP 与 Skills 配置，扩展多语言支持，并进一步收紧文件权限、线程存储和模型流处理边界。

### 会话、图片与 Composer

- 会话支持导出为 Markdown 和 PDF，导出入口整合到会话标题栏，便于归档、分享和继续处理。
- 新增运行时图片上传桥接，画布图片、工作区附件和模型输入可以通过统一的本地文件路径契约传递。
- 生成图片在消息中以轮播形式展示，支持在多张结果之间切换；图片预览灯箱的缩放和拖动边界也得到改进。
- Composer 会按模型分别保存 reasoning effort，切换模型后可以恢复对应的推理强度设置。
- Plan 模式支持单独配置规划模型、Provider 和账号；未设置时自动回退到普通对话模型。
- 单次 Agent 回合的最长运行时间现在可以在高级设置中配置，最高支持 24 小时。

### 项目级 MCP 与 Skills

- 支持在工作区的 `.kun/project.json` 中声明项目所需的 MCP 服务和 Skill 目录。
- Settings -> Agents 新增项目 MCP & Skills 面板，可查看解析后的配置、验证状态、服务器目标和脱敏摘要。
- 项目 MCP 需要用户显式审查并批准；授权绑定规范化配置的 SHA-256 摘要，配置发生语义变化后必须重新审核。
- 项目级路径、Skill 根目录和 stdio 工作目录会进行工作区边界、真实路径和符号链接检查，阻止越界访问。
- 项目配置与用户级 MCP、工作区 Skill、全局 Skill 和已安装扩展 Skill 保持兼容，并支持刷新、授权和撤销。

### 本地化与界面体验

- 新增日语、韩语、俄语、印地语和泰语应用语言，并接入设置、首次启动、IPC 校验和页面语言标记。
- 侧边栏支持拖拽重新排列项目和会话，排序结果会持久化并在重启后恢复。
- UI Plugin 新增 Write、Design 背景插槽、Grand Line 和 Arc Reactor 等场景样式，扩展页面的主题与背景表现更加完整。
- Workflow 编辑器、Write 权限控件和窄窗口布局进一步调整，长模型名称和上下文窗口提示的显示也更加稳定。

### Runtime、存储与权限

- 新增只读、带边界的 Thread Store Doctor，可诊断线程元数据、消息与事件 JSONL、SQLite 索引以及附件完整性。
- 新增带写入互斥、备份、原子替换和稳定性校验的 JSONL 尾部修复能力，只处理可安全确认的损坏末记录。
- 外部路径写入和编辑授权改为按调用、按精确文件范围生效，并重新校验真实文件身份，降低路径替换和符号链接绕过风险。
- 修复上下文压缩重复保留用户指令的问题，避免下一次模型请求重复提交相同内容。
- 修复模型流式工具调用中的待处理参数统计和重复释放问题；检测到状态不一致时会安全失败。
- 修复读取文件的部分行窗口到达文件末尾后仍被错误判定为完整快照的问题，避免合法编辑被 `read-before-edit` 错误拦截。
- 修复 Glob 中 `?` 匹配路径时可能跨越 `/` 的问题，并保留可选 `**/` 的既有行为。
- 修复运行时热更新携带仅在重启时生效的配置字段导致请求被拒绝的问题。
- 配置迁移现在会保留仍然有效的字段，不会因为单个旧字段或无效嵌套字段丢弃整段配置。
- Provider 模型列表支持数组、`data` 和 `models` 等有限返回结构，并对响应体、模型数量和 ID 长度设置上限。
- 空白模型上下文配置默认按 `256K` 处理；Bash 超时改由 Kun Runtime 统一管理，子 Agent 不再重复执行 token budget 限制。
- Windows 检测到 Git for Windows 时，Agent Bash、内置终端和 Workflow Bash 会统一优先使用 Git Bash，并在未安装时保持 PowerShell/cmd 或明确不可用的安全回退。
- 修复 Todo 工具边界处 Plan 关联信息丢失的问题。

### Extension 与发布质量

- 新增扩展包签名载荷规范，绑定扩展身份、版本、发布者、包哈希和有效时间窗口，为后续信任校验提供稳定契约。
- 新增生产依赖审计和 Release 提交校验，并加强扩展索引签名和打包流程检查。
- 新增结构化无障碍标记审计测试，覆盖重复 ID、ARIA 引用、交互控件名称、表单标签和对话框语义。
- 新增可控的 Runtime 故障注入测试框架，覆盖 HTTP 限流、超时、异常 JSON 和 SSE 断连等边界。

### 授权说明

Kun 继续采用非商业许可。企业仅用于内部员工提效时，可通过邮件申请免费的书面内部使用授权；对外 SaaS、托管服务、转售、商业分发及面向客户的产品集成不在该授权范围内。

### 升级说明

- 从 `v0.2.28` 升级可直接通过 GUI 更新，现有工作区、Provider、MCP、Skills 和会话数据不需要手动迁移。
- 项目级 MCP 不会因打开或克隆仓库而自动启动；请在 Settings -> Agents 中检查配置并显式批准。
- `.kun/project.json` 不应提交 API Key、Token、密码或带认证信息的 URL；敏感配置请继续放在用户级 MCP 配置中。
- 使用项目级 MCP 或 Skill 前，请确认项目目录来自可信来源，并检查配置摘要与路径范围。

### 总结

v0.2.29 让 Kun 更适合把会话、图片、项目工具和本地工作区放在同一条工作流中：用户可以导出和复用对话，通过项目配置共享 MCP 与 Skills，并在更多语言和更灵活的 Composer 设置下持续工作。与此同时，线程存储诊断与修复、精确外部路径授权、模型流资源校验和配置迁移加固，为长时间运行和复杂项目提供了更可靠的基础。
